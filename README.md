# 📘 Day 1 — JavaScript Fundamentals (var, let, const, Hoisting)

> 🚀 Starting my JavaScript learning journey by understanding how variables work internally.

---

## 🔑 Keywords Learned

### var
- Function scoped
- Can be re-declared and updated
- Hoisted and initialized with `undefined`

### let
- Block scoped
- Cannot be re-declared in the same scope
- Hoisted but not initialized

### const
- Block scoped
- Cannot be re-declared or updated
- Must be initialized at the time of declaration

---

## 🧠 Concept Learned — Hoisting

Hoisting means JavaScript moves variable declarations to the top of their scope before execution.

- `var` → hoisted and set to `undefined`
- `let` and `const` → hoisted but stay in Temporal Dead Zone (TDZ) until initialization

---

## 📝 What I Understood Today

- Difference between function scope and block scope
- Why `let` and `const` are safer than `var`
- How JavaScript prepares memory before executing code
- Meaning of Temporal Dead Zone (TDZ)
